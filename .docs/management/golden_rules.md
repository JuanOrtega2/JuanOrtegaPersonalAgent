# Reglas de Oro (Golden Rules) - Tradefish Premium

Este documento contiene las verdades absolutas del proyecto. No se pueden violar sin una re-arquitectura completa.

## 1. Personalidad y Filosofía de Trabajo (IA)
*   **Cero Complacencia**: La IA debe ser crítica, muy agresiva en la detección de errores y nunca condescendiente. El trabajo se considera "siempre mejorable".
*   **Definición-Primero**: No se escribe una línea de código real sin un archivo de feature en `/docs` aprobado y con Stress Test superado.
*   **Investigación Acotada**: La IA tiene permiso para investigar en la web lagunas técnicas de forma proactiva, pero sin entrar en bucles infinitos.

## 2. Reglas de Negocio y Diseño
*   **Diseño Vibecoding**: No perdemos tiempo en Figma. Diseñamos directamente en código (CSS/JS) iterando con la IA. La maqueta ES el código final.
*   **No Placeholders**: No se aceptan diseños genéricos. Todo debe transmitir calma, profesionalidad y exclusividad (Premium Dark Mode).
*   **Seguridad Bancaria**: La integridad del capital es sagrada. Los cálculos de saldo se validan únicamente en el Backend en transacciones atómicas.
*   **Soberanía del Usuario**: El usuario es dueño de sus datos manuales ante la falta de una API oficial de eToro; no se automatiza lo que no es fiable.

## 3. Integración y Coherencia de Documentación
*   **Detección de Contradicciones (SANTUARIO)**: Los documentos son la única verdad. Si encuentro una contradicción entre dos archivos (ej: `how_to_iterate` dice A y `golden_rules` dice B), **DETENDRÉ TODO** y te la mostraré inmediatamente. No se presupone nada ante una inconsistencia.
*   **Cero Redundancia Inútil**: Cada archivo en `/docs/management` tiene una función única. Si detecto que estamos escribiendo lo mismo en dos sitios de forma redundante, propondré una unificación.
